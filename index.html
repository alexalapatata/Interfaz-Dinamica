Formulario dinámico


En la página tienes una zona vacía #elementos donde se insertan tarjetas creadas dinámicamente y un formulario oculto (#contenedor_formulario_producto_nuevo) con inputs para imagen, nombre, descripción y precio; un botón fuera del formulario llama a mostrarFormularioAgregarProducto() para mostrar/ocultarlo. En JavaScript manejas un arreglo productos y una clase Producto para almacenar cada producto; obtenerImagen(event) usa FileReader para convertir la imagen subida en una dataURL y previsualizarla, y AgregarProducto(event) crea un FormData, valida los campos y añade un nuevo Producto al arreglo, luego llama a crearTarjeta() que construye el DOM (imagen, título, descripción, precio, botón) y lo inserta en #elementos. También tienes una función crearimagen(event) (usada para ejemplos) que genera N tarjetas repetidas con una imagen fija y texto de muestra. En conjunto el flujo es: mostrar formulario → subir/preview imagen → completar campos → agregar → aparece la tarjeta en la lista dinámica.


API


La página tiene un botón Mostrar/Ocultar y un contenedor #contenedor donde se insertan las tarjetas de Pokémon; originalmente la intención es obtener la lista desde https://pokeapi.co/api/v2/pokemon?limit=100. El script fetch.js hace una petición para la lista y luego solicita los detalles de cada Pokémon (sprites y nombre) para crear tarjetas con la imagen (sprites.front_default) y el nombre, que se insertan en #contenedor. Con el toggle se controla la visibilidad del grid: al mostrarlo se cargan (o se muestran) las tarjetas y al ocultarlo se oculta el contenedor; en la versión corregida se evita ejecutar las peticiones automáticamente al cargar la página y se cargan los datos sólo la primera vez que el usuario abre el contenedor (lazy load), evitando peticiones innecesarias.
